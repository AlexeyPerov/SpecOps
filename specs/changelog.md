# Changelog

## 2026-06-06

- **Chat-http provider and terminology fixes.** Chats UI now derives `chatContextKind` from the active app context (not only chat store scope), so the provider dropdown shows **Debug Provider** (not workspace Debug Agent) and hides **HTTP** until the connection is configured in both Chats and Workspaces. Switching chat scope normalizes in-memory thread providers (`debug-workspace` ↔ `debug-chat`). Settings: **Connections** renamed to **Providers**; debug settings tabs unified to **Debug Provider**. Chat-http uses “New chat”, “Message chat”, and auto-creates a draft chat on send when none is active.
- **Split Debug providers for Chats vs Workspaces.** Replaced the single `debug` provider with scoped ids `debug-chat` (Debug AI, Chats context, ask-only) and `debug-workspace` (Debug Agent, workspace agents, ask+review). Settings now store independent `providerSettings.debugChat` and `providerSettings.debugWorkspace` blocks (shared simulation shape). Settings sidebar: **Chats** → Connections + Debug AI; **Workspaces** → Debug Agent. Chat rail appears when HTTP is configured **or** Debug AI is enabled. Legacy persisted `provider: "debug"` normalizes to the scoped id by chat scope on load.
- **Settings sidebar categories.** Settings dialog sidebar now groups options into top-level **Editor** and **Shortcuts** tabs, with **Chats** and **Workspaces** section labels above **Connections** and **Debug AI** respectively. Section labels use smaller indent, bold 13px type; all selectable tabs share a larger uniform indent so headings are visually distinct from options.
- **Workspace rail reorder.** Workspace icons in the activity rail can be reordered by drag-and-drop (when two or more workspaces are open) or via right-click **Move Up** / **Move Down**. Order is persisted per window in the session snapshot. Active context is unchanged on reorder; new workspaces still append at the end. Added hidden `workspace.reorder` command for automation (`fromIndex` / `toIndex` payload).
